## 快速开始

### 第一次使用

如果你是第一次使用这个框架，需要先创建一个运营账号：

```
"创建一个美食评论账号，名称叫吃货一本经"
```

Skill 会自动：
1. 创建 `吃货一本经/` 文件夹
2. 生成4个配置文件
3. 提示你完善配置

### 已有账号

如果你已经创建了账号，可以直接开始生成内容：

```
"用吃货一本经账号，生成今天的美食快讯"
```

## 创建新账号

### 步骤1：发起创建请求

告诉 Skill 你想创建的账号：

```
"创建一个技术博客账号"
```

### 步骤2：回答引导问题

Skill 会问你几个问题：

1. **账号名称**：给账号起个名字（如"技术博客"）
2. **账号类型**：选择一个类型或自定义
   - 技术博客
   - 营销文案
   - 美食评论
   - 新闻媒体
   - 自定义
3. **账号定位**：一句话描述账号的定位和目标受众
   - 例："为技术人员分享实战经验和架构设计"
4. **启用模块**：选择需要的工作流模块
   - 选题模块（自动从信息源检索内容）
   - 内容生成模块（必须启用）
   - 配图模块（**强烈推荐**：图片必须下载到本地）
   - 质量检查模块（自动检查质量）
   - 内容发布模块（可选，支持微信公众号等平台）

### 步骤3：自动生成配置

Skill 会自动创建：
- `{账号名称}/` 文件夹
- **核心文件（必选）**：
  - `account-config.md`（账号配置总览）
  - `writing-style.md`（写作风格定义）
  - `content-structure.md`（内容板块规划）
  - `content-plan.md`（内容计划与预告管理）
  - `output/current-run-status.md`（执行状态记录）
- **模块文件（根据选配模块生成）**：
  - **选题模块**：生成 `data-sources.md`（信息源配置）和 `output/topic-history.md`（选题历史记录）
  - **发布模块**：生成 `publish-config.md`（发布策略）和 `.env.{账号名称}`（敏感凭证）
- **目录结构**：创建 `output/` 和 `assets/brand-assets/` 目录

### 步骤4：完善配置

根据提示编辑生成的配置文件：
- 特别是 `content-structure.md`，规划你的内容板块

## 规划内容板块

### 为什么需要规划内容板块

内容板块是你账号的内容分类，每个板块有独特的定位、风格和更新频率。

**示例：美食评论账号的板块**
- 美食快讯（每日更新，800-1200字）
- 美食教程（每周2-3次，2000-2500字）
- 美食探店（每周1次，2500-3000字）

### 如何规划内容板块

编辑 `{账号}/content-structure.md`：

```markdown
## 账号板块规划

### 板块1: 美食快讯
- 定位: 每日美食新闻简报
- 字数范围: 800-1200字
- 更新频率: 每日

#### 必选模块
1. **开篇语** (50-100字): 简短开场
2. **热点清单** (5-10条): 当日美食新闻
3. **互动结尾** (50字): 引导评论

### 板块2: 美食教程
- 定位: 实用烹饪教程
- 字数范围: 2000-2500字
- 更新频率: 每周2次

#### 必选模块
1. **菜品介绍** (100字): 菜品背景
2. **食材准备** (200字): 食材清单
3. **制作步骤** (1500字): 详细步骤
4. **小贴士** (200字): 注意事项
```

## 内容规划与预告管理

### content-plan.md - 内容规划文件

**用途**：记录账号的内容预告、连载计划，确保承诺的内容能够兑现。

**为什么需要**：
- 避免承诺无法兑现的内容
- 管理连载内容进度
- 生成前检查，确保能兑现预告

**主要内容**：
1. **已发布的预告**：记录已发布并兑现的预告
2. **待兑现的预告**：记录在文章中承诺但还未发布的内容
3. **连载计划**：记录长期连载内容的进度
4. **内容规划日历**：规划周/月内容

**示例结构**：
```markdown
# 内容规划

## 已发布的预告
- [2026-01-13] 周二将推出"一人食指南" ✅ 已发布

## 待兑现的预告
- [2026-01-14] 周三将推出"外卖精选"
- [2026-01-15] 周四将推出"轻食推荐"

## 连载计划

### 【系列1】网红餐厅测评系列
- 总期数: 10期
- 进度:
  - 第1期: 火烧云傣家菜 ✅ 已发布
  - 第2期: XX餐厅 ⏳ 计划中
  - 第3期: XX餐厅 ⏳ 计划中

## 内容规划日历

### 本周计划
- 周一: 美食快讯 - 车圈热点一览
- 周二: 上班族指南 - CBD午餐地图
- 周三: 网红测评 - XX餐厅
```

**使用流程**：
1. 在文章中添加预告时，同时记录到 `content-plan.md`
2. 生成文章前先查看"待兑现的预告"列表
3. 如无法兑现，及时更新或删除预告
4. 发布文章后，将预告从"待兑现"移动到"已发布"

### 配图管理规范

**重要**：v2.0开始，所有图片必须下载到本地，禁止在文章中使用外部URL。

**图片处理流程**：
```
1. 生成/获取图片（AI生成或网络搜索）
2. 下载到本地（必须）
   - 保存路径: {账号}/output/{日期}/images/
   - 文件命名: {日期}-{板块简称}-{序号}.jpg
3. 创建清单（必须）
   - 文件: images/manifest.txt
   - 记录: 图片来源、用途、时间等
4. 文章引用
   - 使用相对路径: ./images/{文件名}
   - 标注来源: *图片来源: AI生成* 或 *图片来源: XX平台*
```

**图片命名规范**：
- 格式: `YYYY-MM-DD-{板块简称}-{序号}.jpg`
- 示例: `2026-01-13-kuaixun-01.jpg`（快讯第1张图）

**manifest.txt示例**：
```markdown
# 2026-01-13 文章配图清单

## 文章: 2026-01-13-美食快讯-车圈热点一览.md

### 图片1: 美食资讯主题图
- 文件名: 2026-01-13-kuaixun-01.jpg
- 来源: AI生成（豆包）
- 生成时间: 2026-01-13 09:00
- 用途: 文章头图

### 图片2: 餐厅环境图
- 文件名: 2026-01-13-kuaixun-02.jpg
- 来源: 网络下载（大众点评）
- 原URL: https://xxx.jpg
- 下载时间: 2026-01-13 09:05
- 用途: 文章配图
```

**为什么这样规范**：
- ✅ 避免图片URL失效（URL通常只有24小时有效期）
- ✅ 永久保存，便于管理和备份
- ✅ 提升文章加载速度
- ✅ 清晰记录图片来源，便于版权管理

## 编辑账号配置

### account-config.md - 账号配置

**用途**：账号的总览配置

**可编辑内容**：
- 工作流模块开关
- 输出目录和文件命名规则
- 板块列表

### writing-style.md - 写作风格

**用途**：定义账号的写作风格

**可编辑内容**：
- 语气（专业/轻松/正式/口语化/幽默）
- 语言（中文/英文/双语）
- 人设（如：资深美食评论家/邻家吃货）
- 品牌调性（温暖、专业、有趣、犀利等）
- 必须使用/禁止使用的表达
- Emoji使用规则
- 字数标准
- 内容质量标准

**示例**：
```markdown
## 语言风格
- 语气: 轻松、幽默、接地气
- 语言: 中文
- 人设: 邻家吃货，热爱美食，喜欢分享
- 品牌调性: 温暖、有趣、真实

### 表达规范
- 必须使用的表达:
  - 说实话
  - 你懂的
  - 真香
- 禁止使用的表达:
  - 据悉
  - 笔者
  - 综上所述

### Emoji使用规则
- 使用频率: 每500字3-5个
- 推荐Emoji: 🍜 😋 📍 ✅
```

### content-structure.md - 内容板块

**用途**：规划账号的内容板块

**可编辑内容**：
- 板块数量和名称
- 每个板块的定位、字数范围、更新频率
- 每个板块的必选/可选模块

### data-sources.md - 数据源

**用途**：定义信息源和选题策略

**可编辑内容**：
- 监控站点列表
- 选题数量和周期
- 评分标准和维度

**示例**：
```markdown
### 监控站点
1. 美食杰: https://www.meishij.net/
   - 关注方向: 菜谱、美食资讯
   - 更新频率: 每日
   - 适合板块: 美食快讯、美食教程

## 选题策略
- 选题数量: 每次至少3个候选选题
- 选题周期: 每日

## 评分标准
- 维度1: 实用性 - 权重40%
- 维度2: 时效性 - 权重30%
- 维度3: 趣味性 - 权重30%
- 最低通过分数: 25分（满分40分）
```

## 常见问题

### Q: 如何修改账号的写作风格？

**A**: 编辑 `{账号}/writing-style.md` 文件，调整语气、人设、字数标准等。

### Q: 如何添加新的内容板块？

**A**: 编辑 `{账号}/content-structure.md`，添加新的板块定义。

### Q: 如何禁用某个工作流模块？

**A**: 编辑 `{账号}/account-config.md`，在"启用的模块"部分取消勾选。

### Q: 生成的内容不满意怎么办？

**A**: 可以：
1. 调整 `writing-style.md` 中的风格要求
2. 调整 `content-structure.md` 中的结构要求
3. 重新生成

### Q: 可以删除账号吗？

**A**: 可以，直接删除账号文件夹即可。建议先备份重要内容。

### Q: 配置文件有语法错误怎么办？

**A**: Skill 会尝试读取配置，如果格式错误会提示。检查 Markdown 格式是否正确。

### Q: 可以使用自己的风格模板吗？

**A**: 可以，创建账号时选择"自定义"，或编辑生成的配置文件。

### Q: 图片URL有时效性怎么办？

**A**: v2.0开始，所有图片必须下载到本地：
1. 生成或搜索图片后立即下载
2. 保存到 `{账号}/output/{日期}/images/`
3. 在文章中使用相对路径引用：`./images/{文件名}`
4. 创建 `manifest.txt` 记录图片信息

**为什么**：外部URL通常只有24小时有效期，下载到本地可以永久保存。

### Q: 如何管理内容预告？

**A**: 使用 `content-plan.md` 文件：
1. 在文章中添加预告时，同时记录到 `content-plan.md`
2. 生成文章前先查看"待兑现的预告"列表
3. 如无法兑现，及时更新或删除预告
4. 发布文章后，将预告从"待兑现"移动到"已发布"

### Q: 如何发布到微信公众号？

**A**:
1. 在 `account-config.md` 中启用"内容发布模块"
2. 配置微信公众号的 AppID 和 Secret
3. 在配置中设置是否自动发布
4. 生成内容后会自动或手动发布到微信公众号

**注意**：发布平台配置包含敏感信息，请妥善保管。

### Q: manifest.txt 是必须的吗？

**A**: 是的，必须创建。它记录了：
- 图片文件名
- 图片来源（AI生成/网络下载）
- 生成/下载时间
- 图片用途

这个文件有助于：
- 版权管理
- 图片追溯
- 内容审核
