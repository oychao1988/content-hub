## 内容质量标准

### 时效性要求
- 优先使用最近7-30天的网络信息
- 搜索时注意日期标识，避免使用过时信息
- 对于时效性强的内容（如新开业、限时优惠），必须确认信息仍在有效期内

### 事实准确性要求
- 根据账号类型选择合适的表述方式：
  - **线上信息整合类账号**: 使用"网友表示"、"综合XX条评价"、"根据XX报道"等客观表述
  - **线下探店类账号**: 基于真实体验，可使用第一人称描述
- 区分"餐厅官方信息"和"网友评价/媒体报道"
- 数字、数据必须有来源支撑

### 来源标注要求
- **必须**在文章末尾标注信息来源（Sources部分）
- 使用Markdown超链接格式：`-[描述](URL)`
- 网络图片标注"图片来源: XX平台"
- AI生成图片标注"图片来源: AI生成"
- 本地图片标注"图片来源: 本地拍摄"或"图片来源: 官方素材"

### 原创性要求
- 必须是原创整理和观点，禁止直接抄袭
- 引用他人观点时必须标注来源
- 整合多篇信息时要有自己的分析和总结

## 配图要求（强制）

### 图片数量
- 每篇文章至少1-3张配图
- 头图（首图）必须存在

### 图片处理流程
```
1. 生成/获取图片（AI生成或网络搜索）
2. 下载到本地（必须）
   - 保存路径: {账号}/output/{日期}/images/
   - 文件命名: {日期}-{板块简称}-{序号}.jpg
3. 创建清单: images/manifest.txt（记录图片来源、用途）
4. 文章引用: 使用相对路径 ./images/{文件名}
```

### 图片命名规范
- 格式: `YYYY-MM-DD-{板块简称}-{序号}.jpg`
- 示例: `2026-01-13-kuaixun-01.jpg`

### 图片清单文件
每篇文章的images目录必须包含`manifest.txt`:
```markdown
# 2026-01-13 文章配图清单
## 文章: {文章名}
### 图片1: {图片描述}
- 文件名: 2026-01-13-xxx-01.jpg
- 来源: AI生成 / 网络下载（XX平台）
- 用途: 文章头图 / 配图
```

### 禁止事项
- ❌ 禁止在文章中直接使用外部图片URL
- ❌ 禁止使用未经标注来源的网络图片
- ❌ 禁止使用有版权争议的图片

## 特殊要求
- 必须包含价格信息
- 必须包含具体地址（如可获取）
- 必须包含营业时间（如可获取）

## 禁止事项
- ❌ 禁止使用下期预告内容（避免无法兑现）
- ❌ 禁止虚构个人探店体验（线上信息整合类账号）
- ❌ 禁止使用过时的优惠信息、活动信息
